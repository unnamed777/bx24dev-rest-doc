event.offline.list ↗︎

Метод для чтения текущей очереди без вносения изменений в ее состояние, в отличие от event.offline.get. Доступность оффлайн событий можно проверить через метод feature.get.

Параметры

Метод Описание Тип
filter Фильтр записей. По умолчанию отдаются все записи, без фильтрации. Поддерживается фильтрация по полям:
ID,
TIMESTAMP_X,
EVENT_NAME,
MESSAGE_ID,
PROCESS_ID,
ERROR
со стандартными операциями типа =, >, <, <= и т.д.		 array
order Сортировка записей. Поддерживается сортировка по тем же полям, что и в фильтре. На вход принимается массив вида [поле=>ASC|DESC]. По умолчанию - [ID:ASC]. array
startНомер страницы вывода. Работает для https запросов.string
